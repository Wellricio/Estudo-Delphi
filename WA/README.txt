1. Foco no Aprendizado da Linguagem e Ferramentas
"Minha escolha foi desenvolver uma aplicação desktop porque queria aprofundar meu conhecimento em Delphi e Pascal, que são amplamente utilizados no desenvolvimento de software desktop. Meu objetivo principal com este projeto foi explorar o potencial dessas ferramentas e aprender a lidar com desafios como conexão com banco de dados, interfaces gráficas e execução local."

2. Cenário de Uso Ideal para Aplicações Desktop
"Aplicações desktop são mais adequadas em cenários onde os dados precisam ser acessados ou manipulados localmente, sem depender de uma conexão constante com a internet. Um sistema de gestão de registros de ponto, como o que desenvolvi, geralmente é usado em empresas menores que valorizam soluções simples e locais, sem a necessidade de servidores ou infraestrutura web."

3. Facilidade de Implantação e Custo Reduzido
"Uma aplicação desktop tem vantagens para empresas que não possuem equipes de TI robustas ou infraestrutura para manter servidores web. É uma solução prática, de fácil instalação e que pode ser usada imediatamente em máquinas locais, sem custos adicionais com hospedagem."

4. Segurança dos Dados
"A escolha por uma aplicação desktop também garante que os dados sensíveis dos funcionários (como horários de trabalho e folha de pagamento) fiquem armazenados localmente, o que pode ser uma preferência para empresas que querem mais controle sobre suas informações e menos dependência de terceiros para armazenamento na nuvem."

5. Simplicidade no Desenvolvimento Inicial
"Para o propósito deste projeto, desenvolver uma aplicação desktop permitiu um ciclo de desenvolvimento mais direto. Não precisei me preocupar com back-end, front-end e deploy em um servidor, o que me deu mais tempo para me concentrar em criar funcionalidades robustas no core do sistema."

6. Possibilidade de Evolução para Web
"Embora tenha começado como uma aplicação desktop, planejo no futuro expandir para uma solução híbrida ou mesmo totalmente web. Começar pelo desktop foi uma escolha estratégica para consolidar a lógica do negócio e entender bem o funcionamento básico do sistema antes de enfrentar os desafios de uma arquitetura web."
